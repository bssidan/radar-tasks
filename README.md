# רדאר התלויות הניהולי (Radar Tasks)

אפליקציית ניהול משימות ותלויות מבוססת Flutter — **מקומית לחלוטין**, ללא שרת, ללא התחברות, ללא ענן. כל הנתונים נשמרים על המכשיר בלבד.

הרעיון המרכזי: **לא רשימת משימות — אלא ניהול תלויות בין אנשים**. כל משימה ממוקמת על "רדאר" 2x2 סביב "אני", לפי מי מחכה למי.

---

| קטגוריה | משמעות | צבע | מיקום ברדאר |
|---|---|---|---|
| **מחכים לי** | אחרים ממתינים שאני אתקדם | אדום 🔴 | ימין למעלה |
| **בטיפול פעיל** | בידיים שלי, פעיל עכשיו | ירוק 🟢 | ימין למטה |

## מה כלול

- ✅ **רדאר 2x2** עם ספירת משימות, תצוגה מקדימה, והוספה מהירה לכל קטגוריה
- ✅ **אחסון מקומי** עם SQLite (Drift) — מסד נתונים type-safe, כולל היסטוריית שינויים
- ✅ **התראות מקומיות**: תזכורת לכל משימה + תזכורת בוקר יומית חוזרת ("יש לך X משימות שמחכות לך…")
- ✅ **Widget למסך הבית** (Android): ספירת מחכים-לי / בטיפול / באיחור + המשימה הקרובה
- ✅ **מסך "היום שלי"** עם פיצ'ר "התחל יום" (5 המשימות החשובות)
- ✅ **חיפוש, סינון ומיון** בכל מסך
- ✅ **פעולות תקשורת**: חיוג / וואטסאפ / מייל ישירות מאיש הקשר של המשימה
- ✅ **ייצוא / ייבוא JSON** (גיבוי ושחזור, מיזוג או החלפה)
- ✅ **עברית מלאה + RTL**, מצב כהה (Dark navy)
- ✅ **Onboarding** חד-פעמי

## ⚡ בנייה מהירה דרך GitHub Actions (מומלץ — בלי להתקין כלום)

זו הדרך שבחרת. ה-APK נבנה אוטומטית בענן ואתה רק מוריד אותו.

1. צור חשבון GitHub (אם אין) והקם מאגר (repository) חדש — למשל בשם `radar-tasks`.
2. העלה את **כל** תוכן התיקייה הזו למאגר. אפשר דרך הממשק (Upload files) או דרך git:
   ```bash
   cd radar_tasks
   git init
   git add .
   git commit -m "Radar Tasks initial"
   git branch -M main
   git remote add origin https://github.com/USERNAME/radar-tasks.git
   git push -u origin main
   ```
3. ברגע שהקוד עולה ל-`main`, ה-Workflow מתחיל אוטומטית. אפשר גם להריץ ידנית:
   לשונית **Actions** → **Build Android APK** → **Run workflow**.
4. כשהבנייה מסתיימת (סימן ✓ ירוק, בערך 5–10 דקות), נכנסים לריצה:
   - בתחתית העמוד, תחת **Artifacts**, מורידים את `radar-tasks-apk`.
   - או, אם הרצת ידנית, ה-APK יופיע גם תחת **Releases**.
5. מעבירים את קובץ ה-`app-release.apk` לטלפון ומתקינים (ראה "התקנה על הטלפון" למטה).

> ה-Workflow כבר מטפל אוטומטית בשלב הקריטי של יצירת הקוד (`build_runner`) שמייצר את `app_database.g.dart`. אין צורך לעשות כלום ידנית.

## 🛠️ בנייה מקומית (אופציונלי — אם מתקינים Flutter על המחשב)

דורש [Flutter SDK](https://docs.flutter.dev/get-started/install) (גרסה 3.27 ומעלה) + Android SDK.

```bash
cd radar_tasks
flutter pub get

# שלב קריטי — מייצר את app_database.g.dart (Drift):
flutter pub run build_runner build --delete-conflicting-outputs

# הרצה על מכשיר/אמולטור מחובר:
flutter run

# או בניית APK להתקנה:
flutter build apk --release
# הקובץ ייווצר ב: build/app/outputs/flutter-apk/app-release.apk
```

> ⚠️ **חשוב:** בלי הרצת `build_runner` הפרויקט **לא יתקמפל**, כי הקובץ `lib/database/app_database.g.dart` נוצר אוטומטית על ידו (זו דרך העבודה התקנית של Drift). ב-GitHub Actions זה קורה אוטומטית.

## 📱 התקנה על הטלפון (Sideload)

מכיוון שה-APK לא מגיע מחנות Google Play:
1. מעבירים את `app-release.apk` לטלפון (מייל / Drive / כבל USB).
2. בטלפון פותחים את הקובץ. ייתכן שתתבקש לאשר **"התקנה ממקורות לא ידועים"** עבור האפליקציה שדרכה פתחת (למשל הדפדפן / מנהל הקבצים) — מאשרים.
3. מתקינים. בהפעלה ראשונה האפליקציה תבקש הרשאת **התראות** — יש לאשר כדי שהתזכורות יעבדו.

├── theme/app_theme.dart           # ערכת עיצוב כהה

├── widgets/                       # רכיבים לשימוש חוזר (TaskTile, EmptyState)
└── screens/                       # כל המסכים
    ├── splash / onboarding / main_shell
    ├── radar / today / all_tasks / settings
    ├── category / task_details
    └── add_edit_task

android/
├── app/build.gradle               # applicationId, desugaring, multidex
├── app/src/main/AndroidManifest.xml
├── app/src/main/kotlin/.../MainActivity.kt

## ⚠️ מגבלות ידועות

- **iOS עדיין לא ממומש.** הפרויקט בנוי Android-first לפי הבקשה. הוספת iOS תדרוש: WidgetKit extension, App Group, והגדרות הרשאות ב-Info.plist. הקוד המשותף (Dart) כבר מוכן ל-iOS; חסר רק הצד הנייטיב.
- **חתימת ה-APK:** ה-build בנוי עם חתימת debug כדי שיהיה ניתן להתקנה ישירה. להפצה רשמית (Play Store) יש להגדיר keystore משלך ב-`android/app/build.gradle`.
- **התראות מדויקות:** ב-Android 12+ ייתכן שתצטרך לאשר הרשאת "אזעקות מדויקות" בהגדרות המערכת כדי שתזכורות יגיעו בזמן מדויק. חלק מהיצרנים (Xiaomi/Huawei) מצריכים להוסיף את האפליקציה לרשימת "אפליקציות מוגנות" כדי שהתזכורות ישרדו אופטימיזציית סוללה.
- **גרסת Flutter:** הקוד משתמש ב-API מודרניים (`withValues`, `activeThumbColor`). ה-Workflow מקובע ל-Flutter 3.27. בנייה מקומית עם גרסה ישנה מאוד עלולה לדרוש התאמות קלות.
- **אזור זמן:** התזמון מבוסס על אזור הזמן של המכשיר, עם ברירת מחדל Asia/Jerusalem אם הזיהוי נכשל.
